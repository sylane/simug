# Task 7.3: Bootstrap context abstraction

## Objective
- Make bootstrap guidance context optional and discoverable so task bootstrap remains safe when `docs/WORKFLOW.md` or `docs/PLANNING.md` are missing or use a different format.

## Scope
- In: bootstrap prompt guidance discovery, planning scope-lock fallback behavior, prompt/lock regression tests, and design/workflow/changelog updates.
- Out: protocol schema changes, branch policy changes, and broader orchestration-loop modularization.

## Key Decisions
- Discover bootstrap guidance opportunistically from common repo docs (`AGENTS.md`, workflow/planning docs, `README.md`) instead of hard-coding simug-specific file names into prompts.
- Enforce planning-status drift checks only when a discovered planning file both uses the supported task-status format and contains the approved locked task; otherwise execution stays bound by approved intent, branch, and report validation.
