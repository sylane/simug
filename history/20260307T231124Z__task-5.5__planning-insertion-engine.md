# Task 5.5: Planning insertion engine for issue-derived tasks

## Objective
- Insert deterministic TODO planning tasks derived from validated issue triage reports when `needs_task=true`.

## Scope
- In: planning parser/inserter logic, stable task-ID suffix derivation, markdown integrity safeguards, orchestration wiring and tests.
- Out: bootstrap execution of inserted tasks and PR backlinking.

## Key Decisions
- Parsing strategy: use strict markdown task-line regex parsing and fail-closed when no DONE anchor task exists.
- ID strategy: derive inserted task IDs from the last DONE task with alphabetical suffix increments (`a`, `b`, ..., `z`, `aa`, ...) and collision skipping against existing IDs.
- Idempotency strategy: add deterministic source marker line (`Source: issue #<n>`) and reuse existing inserted task ID when same issue/title marker is already present.
- Orchestration wiring: on validated `issue_report.needs_task=true`, insert/reuse planning task and persist `pending_task_id` in state.
