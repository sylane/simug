# Task 7.4d: Verbose progress console and full transcript log

## Objective
- Make verbose mode high-signal instead of prompt-dump heavy, and archive a durable timestamped transcript for each Codex attempt.
- Fix the real-Codex preflight/gate regression where successful `codex` probes are rejected because sandbox-only `~/.codex/tmp/arg0` warnings are treated as fatal.

## Scope
- In: verbose console milestones, per-attempt transcript artifacts, Codex preflight/gate warning classification, related tests/docs/changelog.
- Out: protocol schema changes, new gate suites, unrelated archive format redesign.

## Key Decisions
- `simug run --verbose` now emits concise milestones instead of dumping full prompts; the full prompt/output exchange is preserved in per-attempt `transcript.log` artifacts.
- Codex preflight treats `~/.codex/tmp/arg0` maintenance warnings as non-fatal when the probe itself succeeds, but keeps them fatal when the probe exits non-zero.
- Real-Codex gate scripts print explicit `preflight`, `protocol_canary`, and `recovery_canary` phases so deferred gate work is visibly separate from the commit-producing execution turn.
