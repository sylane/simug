# Task 7.4c: execution-turn protocol discipline and gate decoupling

## Objective
- Reduce protocol echo risk in write-enabled turns and keep commit-producing execution turns deterministic even when environment-sensitive gates must happen later.

## Scope
- In: write-enabled prompt contract changes, execution-turn gate deferral rules, docs/tests/changelog updates
- Out: transcript persistence changes, archive forensic expansion, new protocol schema beyond the bounded envelope from Task 7.4b

## Key Decisions
- Write-enabled prompts now describe the bounded coordinator contract as schema/field rules rather than embedding literal begin/action/end JSON examples, reducing the chance that active-turn payloads are echoed back verbatim.
- Managed PR, bootstrap execution, and repair prompts now explicitly defer environment-sensitive validation gates such as `scripts/canary-real-codex-gate.sh` to follow-up validation after the commit-producing turn finishes.
- Read-only intent/triage prompts keep explicit examples so protocol teaching remains strong where echo risk does not threaten commit-producing turns.
