# Task 3.6: Operator failure explainer

## Objective
- Provide a one-command failure summary with reason, violated invariant, and suggested next action.

## Scope
- In: failure explanation command/flow using events log and archived attempt artifacts.
- Out: changes to core orchestration semantics.

## Key Decisions
- Add CLI subcommand `simug explain-last-failure` to keep diagnostics available without entering the worker loop.
- Build explainer from persisted artifacts only: latest failed `tick_end` event, related failed `invariant_decision`, and linked archive metadata.
- Keep suggestion logic deterministic with simple heuristic mapping from failure text to next-action guidance.
