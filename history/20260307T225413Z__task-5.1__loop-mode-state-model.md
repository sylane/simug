# Task 5.1: Loop-mode state model for issue-first intake

## Objective
- Add explicit persisted orchestration modes and deterministic mode transitions as foundation for issue-first intake.

## Scope
- In: state schema updates (`mode`, `active_issue`, `pending_task_id`) and no-PR loop transition wiring.
- Out: authored-issue discovery/selection and issue triage action execution.

## Key Decisions
- Introduced explicit persisted modes: `managed_pr`, `issue_triage`, `task_bootstrap`.
- Added state normalization rules to infer/default mode safely on load/save and keep managed mode metadata coherent.
- Implemented deterministic no-PR transition flow:
  - managed PR closure -> `issue_triage`,
  - current milestone fallback `issue_triage` -> `task_bootstrap` (until Task 5.2 issue selection exists),
  - bootstrap idle -> back to `issue_triage`,
  - bootstrap PR creation -> `managed_pr`.
- Added integration coverage to assert persisted mode for managed-PR and no-PR-idle flows.
