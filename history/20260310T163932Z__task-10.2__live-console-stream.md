# Task 10.2: Live console stream with channel filtering

## Objective
- Add a console-visible Codex stream so operators can follow simug/Codex interaction in real time without exposing raw coordinator protocol noise by default.

## Scope
- In: CLI/runtime flag plumbing, agent-run streaming behavior, console routing/filtering, and tests/docs tied to operator visibility.
- Out: manager pause/resume control, interactive terminal bridge, or protocol schema changes.

## Key Decisions
- Added an opt-in `simug run --verbose` / `-v` flag instead of changing default console output, so unattended and scripted runs keep their current noise level.
- Switched agent execution from buffered `CombinedOutput` capture to a shared streaming writer that preserves full raw output for parsing/archive while emitting live classified lines to the console.
- Printed full per-attempt prompts plus tagged live Codex output (`codex[manager]`, `codex[protocol]`, `codex[raw]`) so operators can follow both instructions and responses during bootstrap, managed PR, and repair turns.
