# Task 7.3e: Rebase/squash-merge aware post-merge transition handling

## Objective
- Allow no-PR intake to transition off a just-merged managed branch after GitHub confirms merge, even when the local branch tip is not an ancestor of `origin/main`.

## Scope
- In: merged-PR transition handling, branch-cleanup readiness checks, regression tests, design/changelog/planning updates.
- Out: non-merged branch policy changes, issue-finalization semantics, bootstrap execution behavior.

## Key Decisions
- Carry the merged PR head branch from inactive-PR detection into no-PR intake so readiness checks can distinguish the exact just-merged managed branch from unrelated local branches.
- Allow the ancestry check to be skipped only when the current branch exactly matches the GitHub-confirmed merged managed PR head branch; all other non-`main` branches remain blocked unless they are already merged into `origin/main`.
