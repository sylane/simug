# Task 7.4b: bounded coordinator envelope and turn identity

## Objective
- Replace free-floating `SIMUG:` action parsing with a bounded active-turn envelope keyed by explicit turn identity.

## Scope
- In: coordinator protocol parsing, runtime prompt contract, active-turn validation, protocol-facing tests, design/workflow sync
- Out: transcript archival redesign, verbose-console reduction, later forensic hardening beyond this task

## Key Decisions
- Coordinator machine output now uses bounded `SIMUG:` begin/action/end envelopes keyed by a generated per-attempt `turn_id`, with optional `session_id` matching when simug resumes an existing Codex session.
- Only action payloads inside the matching active envelope count toward terminal/action cardinality; malformed or stale `SIMUG:` lines outside the active envelope are ignored fail-closed instead of affecting the current turn.
- Simug now injects the active turn identity into the prompt and process environment so deterministic tests can emit matching envelopes without hard-coding runtime-generated ids.
