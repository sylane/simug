# Task 6.1: Single-task self-host run mode

## Objective
- Add a one-shot execution mode (`simug run --once`) that runs one tick, persists state, and exits with explicit status codes suitable for supervisor loops.

## Scope
- In: CLI flag parsing, orchestrator run-loop control, deterministic exit-status mapping, and tests.
- Out: supervisor wrapper script and restart journals.

## Key Decisions
- Runtime API: keep `Run` behavior unchanged and add `RunOnce` wrapper that executes one tick, saves state, and exits without waiting for poll interval.
- CLI contract: support `simug run --once` via explicit run-option parser and usage validation (`parseRunArgs`).
- Exit-code policy: use deterministic status codes (`0` success, `2` runtime failure, `64` usage error) for supervisor-friendly process control.

## Residual Risks
- `main()` exit-path behavior is exercised indirectly through `runMain` tests; full process-level signal/exit integration remains best validated by wrapper-level smoke checks (Task 6.2).
