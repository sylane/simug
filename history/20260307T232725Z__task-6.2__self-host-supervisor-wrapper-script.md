# Task 6.2: Self-host supervisor wrapper script

## Objective
- Add a repeatable wrapper script that rebuilds `simug`, executes one-shot loops, captures artifacts/logs, and restarts deterministically from persisted state.

## Scope
- In: shell supervisor script, operator usage docs, and basic script validation tests.
- Out: crash-safe attempt journals and restart state-machine internals.

## Key Decisions
- Wrapper location: add executable `scripts/self-host-loop.sh` as the single entrypoint for self-host supervisor loops.
- Loop contract: each iteration rebuilds `bin/simug`, runs `./bin/simug run --once`, snapshots `.simug/state.json`, and writes per-iteration logs under `.simug/selfhost/<timestamp>/`.
- Exit behavior: wrapper stops immediately on first non-zero simug exit code to preserve deterministic supervisor semantics.

## Validation
- Baseline: `GOCACHE=/tmp/go-build go test ./...` -> pass.
- Focused: `bash -n scripts/self-host-loop.sh && GOCACHE=/tmp/go-build go test ./cmd/simug ./...` -> pass.
- Final: `GOCACHE=/tmp/go-build go test ./...` -> pass.
- Coverage: `GOCACHE=/tmp/go-build GOEXPERIMENT=nocoverageredesign go test ./... -coverprofile=coverage.out && GOCACHE=/tmp/go-build go tool cover -func=coverage.out` -> pass (`total: 65.8%`).

## Residual Risks
- Wrapper currently snapshots state/logs but does not yet persist in-flight attempt journal semantics (covered by Task 6.3).
