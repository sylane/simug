# Task 5.2: Authored-issue discovery and deterministic selection

## Objective
- List open issues authored by the authenticated user and deterministically select a candidate before normal task bootstrap.

## Scope
- In: GitHub issue listing integration, no-PR issue-first selection flow, deterministic ordering tests.
- Out: issue triage prompt/protocol (`issue_report`) and issue comment posting.

## Key Decisions
- Selection policy: choose the lowest issue number from authored open issues for deterministic oldest-first behavior.
- Filtering policy: treat entries with a non-null `pull_request` field as PRs and exclude them from issue selection.
