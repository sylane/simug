# Task 7.5: Single-source prompt contract

## Objective
- Centralize shared SIMUG prompt-contract instructions and examples so runtime prompt builders and prompt regression tests consume one renderer.

## Scope
- In: `internal/app` prompt contract helpers, runtime prompt builders, prompt regression tests, task bookkeeping docs
- Out: protocol behavior changes, parser changes, new coordinator actions

## Key Decisions
- Added a small `promptContractSection` renderer in `internal/app` so prompt builders and prompt regression tests can share the same contract fragments without changing the prompt contract wording.
- Kept mode-specific contract variants separate (`managed PR`, `issue triage`, `bootstrap intent`, `bootstrap execution`, `repair`) so each prompt still controls its exact allowed actions and example payloads.
