# Task 7.6: Idempotent GitHub mutation primitives

## Objective
- Consolidate repeated issue-comment marker dedupe and posting logic into reusable helpers without changing mutation behavior.

## Scope
- In: orchestrator issue-comment idempotency helpers, regression tests, changelog/planning/history updates.
- Out: protocol changes, GitHub API surface changes, issue-link semantics.

## Key Decisions
- Centralize issue-comment idempotency around one helper that owns three responsibilities only: list issue comments, detect same-user markers, and post only when no matching marker exists.
- Route all issue-comment mutation call sites through the same helper, including later-added update/finalization flows, so the repo does not re-diverge after the initial triage/backlink extraction target.
