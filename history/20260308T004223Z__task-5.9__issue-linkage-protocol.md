# Task 5.9: Issue linkage protocol for implementation turns

## Objective
- Extend simug/Codex protocol so Codex can declare issue linkage (`fixes`, `impacts`, `relates`) during normal implementation turns.

## Scope
- In: protocol action model, parser validation, orchestrator prompt contract, mode validation tests.
- Out: persistent issue ledger, issue comment side effects, merge-time issue closure.

## Key Decisions
- Added a new `issue_update` protocol action with strict fields: `issue_number`, `relation` (`fixes|impacts|relates`), and non-empty `comment`.
- Kept issue triage mode strict: `issue_update` remains invalid there; linkage intent is validated for managed/bootstrap turns only.
- Accepted `issue_update` in action application as orchestrator-owned intent only (no direct issue mutation yet), preparing the next tasks for ledger/comment/closure behavior.
