# Task 5.3: Issue triage prompt and protocol extension

## Objective
- Add an explicit issue-triage Codex prompt path and strict `issue_report` protocol handling that the orchestrator can validate and consume deterministically.

## Scope
- In: agent protocol parsing for `issue_report`, no-PR issue-triage execution path, mode-compatible action validation, prompt contract and integration tests.
- Out: posting issue analysis comments, planning insertion, and issue-to-PR backlinking.

## Key Decisions
- Protocol model: add explicit `issue_report` action parsing in `internal/agent` with required typed fields (`issue_number`, `relevant`, `analysis`, `needs_task`) and optional task metadata.
- Validation model: enforce issue-triage mode compatibility in orchestrator (`exactly one issue_report`, report-before-terminal ordering, selected issue match, non-empty analysis, `needs_task=true` requires `task_title` + `task_body`, and no `done changes=true`).
- Repair model: extend shared agent validation loop with mode-specific semantic validator callback so triage protocol violations use the same bounded repair path as other validation failures.
