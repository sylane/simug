# Task 7.3c: Bootstrap execution single-commit fail-closed guard

## Objective
- Make bootstrap execution fail closed unless exactly one commit is produced from the staged baseline, and prevent repair/recovery from continuing when a failed bootstrap attempt already advanced `HEAD`.

## Scope
- In: bootstrap execution validation, repair/recovery decision logic, regression tests, design/changelog updates
- Out: inline review comment handling, merged-branch cleanup

## Key Decisions
- Enforce the exact-one-commit rule in bootstrap execution validation, not after push preparation, so multi-commit outcomes fail before any GitHub mutation path runs.
- Abort automatic repair/replay whenever a failed bootstrap attempt already advanced `HEAD`, both inside the active repair loop and during restart recovery, because continuing from an unknown partial execution state would stack commits silently.
