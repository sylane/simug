# Task 6.3: Crash-safe in-flight attempt journal

## Objective
- Persist in-flight Codex attempt context before and after invocation so restarts can recover deterministically.

## Scope
- In: state schema, run-loop journal persistence hooks, tests, docs/changelog/planning updates.
- Out: recovery decision state machine logic (Task 6.4).

## Key Decisions
- Persist in-flight attempt journal directly in `.simug/state.json` (`in_flight_attempt`) so supervisor snapshots and restart logic read one canonical source.
- Journal writes happen both at attempt start and after attempt result (validated or failed), using immediate state saves to survive crashes between ticks.
- Journal is cleared only after successful post-agent validation, preserving failure context for restart diagnostics.
