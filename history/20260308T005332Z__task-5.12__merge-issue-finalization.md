# Task 5.12: Close-on-merge issue finalization workflow

## Objective
- Finalize tracked issue links when a managed PR is merged, with idempotent comments and closure behavior.

## Scope
- In: merged PR detection path, finalization comment/close flow, idempotency handling, tests/docs/changelog updates.
- Out: broader queue policies beyond same-user tracked issue links.

## Key Decisions
- Inactive managed PR transitions now fetch PR state first and only run issue finalization when the PR is actually merged.
- Merge finalization uses deterministic per-link markers (`simug:issue-finalize:v1`) and same-user scope checks, mirroring implementation-time idempotency principles.
- `fixes` links trigger close-on-merge only when the issue is still open; `impacts`/`relates` links post informational finalization comments without closing.
