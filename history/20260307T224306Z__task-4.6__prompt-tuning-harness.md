# Task 4.6: Prompt tuning harness for repeatable protocol failures

## Objective
- Add repeatable harness tests for common Codex failure patterns and repair-loop prompt behavior.

## Scope
- In: deterministic failure corpus tests and bounded repair-loop behavior for parse/validation failures.
- Out: full session-runtime redesign.

## Key Decisions
- Extend `runAgentWithValidation` so Codex execution/protocol errors also use the bounded repair loop (instead of failing immediately).
- Add repeatable harness tests for three corpus classes:
  - protocol failure that recovers on repair prompt,
  - validation failure (dirty tree) that recovers on retry,
  - persistent protocol failure that hard-fails after bounded attempts.
- Keep harness deterministic by fixing `MaxRepairAttempts=1` in tests.
