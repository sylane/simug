# Task 6.6: Live GitHub + real Codex dry-run on a sandbox repo

## Objective
- Provide an executable and auditable sandbox dry-run workflow for real GitHub + real Codex validation across issue-driven and planning-driven lifecycles.

## Scope
- In: sandbox dry-run script, evidence runbook, script contract tests, docs/changelog/planning/history updates.
- Out: self-host end-to-end canary orchestration loop (Task 6.7).

## Key Decisions
- Implemented a sandbox dry-run evidence verifier script instead of hardcoding environment-specific repository assumptions.
- Verification checks are based on merged PR metadata (`gh pr view`) plus optional closed-issue assertion, then written as summary artifacts for audit.
- Added a dedicated runbook with prerequisites and failure handling so execution is reproducible by operators.
