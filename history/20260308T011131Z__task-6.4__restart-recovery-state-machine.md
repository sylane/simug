# Task 6.4: Restart recovery state machine

## Objective
- Implement deterministic restart recovery transitions from persisted in-flight attempt journal state.

## Scope
- In: recovery transition model, invariant checks, state persistence/logging, tests, docs/changelog/planning updates.
- Out: chaos/failure-injection matrix expansion (Task 6.5+).

## Key Decisions
- Recovery actions are deterministic and persisted as `last_recovery` in state for auditability and restart diagnostics.
- Recovery runs before normal tick execution and enforces invariant checks (current branch + clean tree) before choosing `resume`/`replay`/`repair`/`abort`.
- `replay` and `repair` both set `cursor_uncertain=true` to force conservative context replay after restart; `abort` preserves journal context and fails closed.
