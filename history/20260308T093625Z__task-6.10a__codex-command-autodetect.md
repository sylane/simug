# Task 6.10a: Codex command auto-detection + non-interactive defaults

## Objective
- Make simug work out of the box with environment-configured Codex by defaulting to non-interactive invocation without requiring manual `SIMUG_AGENT_CMD` setup.

## Scope
- In: default agent command resolution, real-Codex canary script defaults, related docs/tests/changelog updates.
- Out: Codex account/auth provisioning ownership, session/app-server runtime migration.

## Key Decisions
- Default to `codex exec` when available, because simug interaction is stdin/stdout non-interactive and `codex` interactive mode is not reliable for canary automation.
- Keep explicit overrides first-class (`SIMUG_AGENT_CMD`, canary `--cmd`) so project-specific or profile-specific Codex setup remains user-controlled.
- Keep compatibility fallback to `codex` when `codex exec` probing fails, to avoid breaking older/partial Codex CLI installs.
