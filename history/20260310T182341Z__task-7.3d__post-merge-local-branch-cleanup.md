# Task 7.3d: Post-merge local branch cleanup and transition validation

## Objective
- Delete merged managed local branches during no-PR intake without weakening the existing merge validation, checkout, and main-sync invariants.

## Scope
- In: no-PR intake branch transition path, git helper for local branch deletion, deterministic tests, design/changelog updates
- Out: bootstrap execution validation, review comment context handling

## Key Decisions
- Delete the merged local branch only after the existing merged-into-`origin/main` check, checkout to `main`, and fast-forward sync all succeed, so cleanup cannot bypass the current fail-closed readiness checks.
- Keep branch deletion in `ensureMainReady` instead of the merged-PR detection function, because the invariant belongs to the no-PR intake readiness path and should also apply when simug restarts on a previously merged branch.
